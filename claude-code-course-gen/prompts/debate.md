# Idea 辩论：质疑者 vs 辩护者

你需要对一组教学多媒体 idea 进行内部辩论，判断每个 idea 是否值得生成。

## 输入

Ideas 列表（JSON）：
```
__IDEAS_JSON__
```

## 辩论规则

对每个 idea，执行以下辩论流程：

### 质疑者角色
从以下角度提出 2-3 个质疑：
1. **教学有效性**：这个 animation/game 是否真的比纯文字说明更有效？学生是否真的需要看到动画才能理解？
2. **技术可行性**：在单个 HTML 文件 + Canvas/DOM 的约束下，能否在合理复杂度内实现？是否超出 iframe 渲染能力？
3. **认知负荷**：视觉元素是否过多导致分散注意力？是否存在不必要的花哨效果？
4. **内容准确性**：动画/游戏是否可能传达错误的科学概念？简化是否过度？

### 辩护者角色
针对每个质疑，从以下角度回应：
1. 引用教育心理学原理（如双重编码理论、主动学习理论）
2. 提出具体的实现方案来解决技术顾虑
3. 说明如何控制认知负荷
4. 说明如何保证科学准确性

### 判决
综合双方论点，给出 verdict：
- **pass**：值得生成，质疑已被充分回应
- **fail**：不值得生成，质疑无法被充分回应
- **revise**：需要修改 idea 后再生成（附带修改建议）

## 输出格式

严格输出以下 JSON（不要包含 ```json 标记，直接输出纯 JSON）：

{
  "results": [
    {
      "idea_id": "anim_001",
      "verdict": "pass",
      "challenges": [
        "质疑1: ...",
        "质疑2: ..."
      ],
      "responses": [
        "回应1: ...",
        "回应2: ..."
      ],
      "final_reasoning": "综合判断理由（2-3句话）",
      "revision_notes": ""
    }
  ]
}

## 注意事项

- 质疑必须是实质性的，不能走过场
- 辩护必须直接回应质疑，不能避重就轻
- verdict 为 revise 时，revision_notes 必须给出具体修改方向
- exercise 类型的 idea 通常直接 pass（练习题实现简单且教学价值明确）
- 质疑和回应都用中文
