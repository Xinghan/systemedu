# Resource Quality Evaluation Rubric

## Relevance Score (1-5)

| Score | Meaning |
|-------|---------|
| 5 | Directly and completely covers this node's topic |
| 4 | Covers the topic with minor tangents |
| 3 | Partially covers the topic; learner would need other sources |
| 2 | Only briefly mentions the topic in a broader context |
| 1 | Barely related; coincidental keyword match |

## Quality Score (1-5)

| Score | Meaning |
|-------|---------|
| 5 | Authoritative source, clear structure, current (< 3 years), interactive or well-illustrated |
| 4 | Trustworthy source, mostly clear, reasonably current |
| 3 | Acceptable quality but may have gaps, ads, or older content |
| 2 | Low production quality, unclear explanations, or outdated |
| 1 | Spam, clickbait, severely outdated, or misleading |

## High-Quality Source Signals

**Web:**
- Official documentation (MDN, Python Docs, React Docs, etc.)
- University course pages (MIT OCW, Stanford Online, Coursera, edX)
- Established tutorial sites (freeCodeCamp, The Odin Project, css-tricks, real-world-oriented)
- Well-maintained GitHub repositories with good documentation
- Recent blog posts from recognized practitioners

**YouTube:**
- Channels with educational focus and consistent production quality
- Videos with high view count relative to upload date
- Clear chapter markers / timestamps
- Instructor has verifiable credentials or strong community reputation

## Red Flags (discard immediately)

- Paywalled content with no free preview
- URLs that return 404 or redirect loops
- Content farms with thin AI-generated text
- Videos that are pure ads or promotional material
- Tutorials for severely outdated versions (e.g., Python 2, Angular 1)
