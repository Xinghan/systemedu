C1 · OpenAQ 一条记录长啥样

measurements JSON schema · what each row carries
FIELD TYPE MEANING REQ
sensor_id int 你的传感器编号 REQ
parameter str 测哪个 · pm25 / pm10 / o3 REQ
value float 浓度数字 REQ
unit str µg/m³ 或 ppb REQ
datetime iso8601 UTC 时间, 别用本地时 REQ
coordinates obj lat / lon · 你的节点位置 REQ
provider str 谁送的 · "PurpleAir" AUTO
isMobile bool 固定点还是移动的 OPT
为什么这些必填: 没 datetime 就排不了时序; 没 coordinates 就上不了地图; 没 unit 就不知道是 µg/m³ 还是 mg/m³ 差 1000 倍。 why required: no datetime → no timeseries · no coordinates → no map · no unit → wrong magnitude.

C2 · 验证你的数据真的到了

GET request · check by your sensor_id
01

02
向 OpenAQ 发一条 GET 查询 Send one GET request to OpenAQ
$ curl -H "X-API-Key: $TOKEN" \   https://api.openaq.org/v3/sensors/12345/measurements
03
看返回的 JSON · 有最近 1 小时数据 = pipeline 跑通了 If JSON has last-hour data → pipeline is alive
