Abstract
摘要 · 4 句话 4 sentences
Q问题:1 句Problem: 1 sent
M方法:1 句Method: 1 sent
R结果:1 句带数字Result: 1 with number
C结论:1 句Conclusion: 1 sent
城市低成本 PM2.5 监测精度不足 (问题)。本文用一台 PurpleAir 节点在阳台采集 30 天数据,与国控站 6 km 外配对回归 (方法)。校正后 MAE 从 8.4 → 3.1 µg/m³,R²=0.86 (结果)。结果表明个人节点经线性校正后可达准公开水平 (结论)。 Low-cost urban PM2.5 sensing lacks accuracy (problem). We deployed one PurpleAir node on a balcony for 30 days, pair-regressed against a reference station 6 km away (method). Post-calibration MAE dropped from 8.4 → 3.1 µg/m³, R²=0.86 (result). Linear calibration brings personal nodes near public-grade accuracy (conclusion).

Introduction
引言 · 背景+缺口+我 Background + Gap + Mine
B大背景:为啥这事重要Why this matters
G缺口:别人还没做啥Gap: what's missing
I我:本文做了啥贡献I: what we did
背景: PM2.5 长期暴露与心血管病显著相关; 中国 1339 个国控站平均间距 25 km。缺口: 既有研究多聚焦站点本身, 对 1-10 km 内城市微尺度差异覆盖不足; 现有低成本传感虽多, 但缺乏与本地参考站的系统配对评估。本文: 我们用单台 PurpleAir 节点 + 30 天连续观测 + 6 km 外国控站作参考, 评估线性校正对 MAE/R² 的提升幅度, 并公开数据集与代码。 Background: chronic PM2.5 exposure links to cardiovascular disease; China's 1339 reference stations average 25 km apart. Gap: prior work focuses on station fidelity, underserving 1-10 km urban micro-scale; low-cost sensor literature lacks systematic paired evaluation against local references. Ours: we deploy one PurpleAir node + 30-day stream + a 6 km reference station, measuring MAE/R² gain from linear calibration, with dataset and code released.

Methods
方法 · 数据+公式+评估 Data + Formula + Eval
D数据集:多大/多久Dataset: size/range
F公式:y=a·x+bFormula: y=ax+b
E评估:MAE/R²/留出Eval: MAE/R²/holdout
数据集: PurpleAir PA-II, 阳台离地 8 m, 2026-04-01 ~ 04-30, 2 分钟一条, 共 21,420 行有效记录。参考: 国控站 6 km 外, 小时均值。校正模型: y = a·x + b, 其中 x = PA 原始 PM2.5, y = 站点真值; 用 70% 训练 30% 留出。评估: MAE、RMSE、R²; 时序图与残差散点用于诊断。工具: Python 3.12 + pandas + scikit-learn; 代码 GitHub 公开, 数据 Zenodo 存档带 DOI。 Dataset: PurpleAir PA-II on balcony 8 m AGL, 2026-04-01 to 04-30, 2-min cadence, 21,420 valid rows. Reference: nearest station 6 km away, hourly mean. Calibration: y = a·x + b, x = raw PA PM2.5, y = reference; 70/30 train/holdout. Evaluation: MAE, RMSE, R²; time series + residual scatter for diagnosis. Tools: Python 3.12 + pandas + scikit-learn; code on GitHub, dataset archived on Zenodo with DOI.

Discussion
讨论 · 意义+局限+下一步 So-what + Limit + Next
S意义:这数字代表啥So-what: what it means
L局限:别人能批啥Limit: what to attack
N下一步:还可以做啥Next: what's after
意义: 单台校正节点已达到 R²=0.86, 接近世卫指南 24 h 限值±15% 区间, 个人健康决策足够用。局限: (1) 仅 30 天数据, 未覆盖冬季供暖; (2) 单点对单参考, 未做空间外推; (3) 线性模型未引入湿度修正, 雨天残差偏大。下一步: 扩展到 5 节点网格 + 引入 RH/T 协变量 + 跑 90 天看季节漂移。 So-what: a single calibrated node already reaches R²=0.86, near WHO 24-h ±15% — adequate for personal health calls. Limits: (1) only 30 days, no winter heating; (2) single node vs single reference, no spatial extrapolation; (3) linear model lacks humidity correction, rainy-day residuals bias up. Next: scale to a 5-node grid + add RH/T covariates + run 90 days to catch seasonal drift.
新手常犯 · 自查清单 Common pitfalls — self check
INTRO
把 Intro 写成 我的故事:"我从小喜欢空气"→ 评审 30 秒走人 Intro as my life story: "I always loved air" — reviewer leaves in 30 s
METHODS
漏公式 + 不写采样频率:别人无法复现 = 自动 reject No formula + no sample rate: not reproducible = auto reject
RESULTS
只说 "有显著提升" 没数字:数字才有信用 "Significant improvement" without numbers: numbers earn trust
DISCUSSION
不写局限 + 吹爆自己:评审一眼就会找漏洞 No limits + self-hype: reviewers will find holes anyway
