把实时解码封成一个稳定模块
前面零散的脚本，整理成一个有清晰接口的实时解码模块：喂进脑电，吐出意图。不管谁来调、调多少次，它都稳稳运转。
S4 · CLIMAX
模块 API 设计 · 5 条接口 RealtimeDecoder
输入接口feed(samples)
接收一小段脑电样本（一个窗口），是模块唯一的数据入口。调用方只管喂，不用懂里面怎么算。
输出接口predict() -> intent
吐出一个意图标签加置信度，比如「左 / 右 / 不动 · 0.87」。格式固定，游戏端拿来就能用。
校准接口calibrate(data)
每个人脑电不一样。开局先校准，让模块认识「这个人的左和右」，之后才准。
错误处理on_error()
信号断了、电极掉了、数据乱了——模块不崩，而是返回明确错误，让游戏端知道该暂停还是提示。
状态查询get_state()
随时能问模块「你现在在干嘛」——空闲、校准中、运行中还是出错。游戏端据此显示对应界面。
运行状态机 get_state()
running · 运行中
校准完成后进入运行：每来一个窗口就 predict 一次，源源不断把意图喂给游戏。
