用所有数据拟合 + 用所有数据评估 = 自欺欺人 Fit on ALL data + evaluate on ALL data = self-deception
"我看到我抄过的题答对了" — 这叫过拟合 (overfit) 加数据泄漏 (data leakage)，不是真本事。 "I got the answers right on questions I already saw" — that's overfit plus data leakage, not real skill.
SYMPTOM A
曲线穿过每个点 Curve passes every point
R² 看着 = 1，像把每道题答案都背下来。新一天数据来了，预测就崩。 R² looks like 1, as if memorising every answer. New data arrives — predictions collapse.
SYMPTOM B
真实新数据来时崩了 Crashes on unseen data
曲线只贴住老点。明天传感器读出的 PM2.5 偏离十几微克，预测全错。 Curve only hugs old points. Tomorrow's reading drifts a lot — predictions go wrong.
PIPELINE / 正确做法 把 30 天数据分两份，公式只看其中一份 Split 30 days into two parts, formula sees only one
1
按时间切 80 / 20 Split 80 / 20 by time
前 24 天 = 训练。最后 6 天 = hold-out 测试 (锁起来)。 First 24 days = train. Last 6 days = hold-out test (locked away).
2
只用训练集拟参数 Fit params on train only
解三参数 a/b/c。测试集这一步绝不能碰。 Solve for a/b/c. Test set must not be touched here.
3
在测试集算 MAE / R² Score MAE / R² on test
把固定后的公式套上测试集，算误差。这是公正成绩。 Run the locked formula on test, compute error. That is the fair score.
4
训练 ≈ 测试 → 没过拟合 Train ≈ Test → no overfit
两边 MAE 接近 → 公式真学到规律。差距大就重练。 Close MAE on both → formula learned the pattern. Big gap → retrain.
随机切Random split NO
测试点夹在训练点里，公式偷偷"看过未来"，分数虚高。 Test points sit between train ones; the formula peeks at the future.
时序切 (用这个)Time split (use this) USE
前 24 天训，最后 6 天测。模拟真实部署：明天的数据我看不到。 Train on first 24d, test on last 6d. Mirrors real deployment.
交叉验证K-fold CV PRO
切 5 份轮流当测试。数据少时更稳。时序版叫 TimeSeriesSplit。 Rotate 5 folds as test. Stabler on small data. Time variant: TimeSeriesSplit.
我永远留 hold-out I always keep hold-out
最后 6 天数据写脚本时就分走，存在另一个文件里。 Set aside last 6 days in a separate file from day one.
我报告测试集指标 I report test-set metrics
README 写的 MAE / R² 必须是测试集上的成绩，不是训练集。 MAE / R² in README must come from test set, not train.
我不偷看测试集 I do not peek at test
不靠测试集结果回去调公式，否则等于又把测试集变训练集。 No tuning the formula based on test scores — that leaks test into train.
