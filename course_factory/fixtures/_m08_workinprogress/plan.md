# 电与焊接安全：万用表 5 分钟入门 + 电平转换概念

## 学习目标

- 能够说出**电压 (V)**、**电流 (A)**、**电阻 (Ω)** 三个物理量是什么，以及它们之间的关系 (欧姆定律)
- 能够用万用表测三件事: **电池电压**、**线路通断**、**电阻阻值** (旋钮 → 表笔 → 读数)
- 能够指出**烙铁安全**的 5 个关键点 (站立位、温度、左手位置、烫伤急救、收纳)
- 能够解释为什么 **5V 数字信号直接接 3.3V GPIO 会烧引脚**，并能用两个电阻搭一个 5V→3.3V 分压电路，万用表验证输出
- **本节验收作品**: 焊接纪念章照片 + 万用表读数截图 + 分压电路示意图 (照片或手绘)

## 引入: 第一次烧板子的故事

M07 你学完了 Linux 命令行, 现在 Pi 是台你能远程控制的小电脑了。下一步你想干嘛? **当然是给它接传感器**——M11 你就要接第一个 PMS5003 PM2.5 颗粒物传感器, 让 Pi 真正能"闻"空气了。

但是先慢着。我给你讲个故事。

某个孩子接的第一个传感器叫 DHT22 (温湿度传感器), 模块上写 "VCC 接 3.3-5V"。他看到 Pi 有 5V 引脚, 顺手接了 5V VCC, 然后传感器的 DATA 引脚也接到了 Pi 的 GPIO17。**没两秒, Pi 黑屏了**。重启发现 GPIO17 永远是高电平, 接什么传感器都不响应。**整块 Pi 报废了**——准确说, 是那个 GPIO 烧穿了。

为什么? 因为 **Pi 的 GPIO 引脚只能承受 3.3V**。传感器收到 5V 供电, 输出的 DATA 信号也是 5V 高电平。**5V 灌进 3.3V 的引脚, 就像给小水管灌大水, 直接撑破**。

这一节我们不接传感器, 但你要先掌握三件事——**测电 (万用表)、焊接 (烙铁)、降压 (电平转换)**, 否则 M11 你重蹈那个孩子的覆辙。

## 核心概念一: 万用表三个旋钮就够用

万用表 (multimeter) 是电子工程师的"温度计 + 体重秤", 任何时候不知道电路对不对, 先掏出它来测。

[[THEORY:theory_voltage_current_resistance]]

便宜的万用表 (淘宝 30-80 块) 一个旋钮转几圈, 但**你只需要懂三个挡位**:

| 挡位 | 标识 | 测什么 | 怎么测 |
|------|------|--------|--------|
| **直流电压** | V⎓ 或 DCV | 电池有多少伏 | 红表笔接 +, 黑表笔接 -, 读屏 |
| **通断 + 二极管** | 喇叭符号 🔊 | 这根线通不通 / 这两点连着没 | 两个表笔分别戳两端, 通了万用表"嘀"一声 |
| **电阻** | Ω | 电阻多少欧 | 表笔分别接电阻两端, 读屏 (注意: 测电阻时电路必须断电!) |

[[IDEA:anim_multimeter_three_dials]]

第一次拿到万用表, **先用它测一节干电池** (1.5V), 旋钮转到 V⎓ "20V" 档 (留余量), 红黑表笔分别接电池正负极。屏幕显示 1.50 或 1.45 左右——这就是这节电池剩多少电。

## 核心概念二: 烙铁安全的 5 件事

焊接 (soldering) 是把两根金属用熔化的锡焊在一起, 关键工具是**烙铁** (soldering iron)——一头是 300-400°C 的发热头, 一头是握把。

这是真烫的东西, 第一次用之前先记住 5 件事:

1. **永远把烙铁放回**烙铁架** (iron stand)**, 不要直接放桌上——烫坏桌子 + 容易碰到。
2. **温度调到 350°C 左右** (太冷锡不化, 太热烫坏元件), 烙铁头先在湿海绵上擦一下 (清洁 + 检查温度)。
3. **左手永远不要伸到烙铁头前方**——焊接时元件会从板子另一边戳出来, 你左手扶着元件, 但**手指要离焊点至少 3 cm**。
4. 万一**烫到手, 第一时间冷水冲 10 分钟**——不要涂牙膏 / 不要涂酱油, 这些都是网上的胡说。
5. **焊接时窗户开一条缝**——焊锡蒸汽含铅, 长时间不通风不健康。

[[THEORY:theory_solder_basics]]

[[IDEA:image_soldering_safety_grip]]

第一次练焊, 找一块**废 PCB 板** (或淘宝几块钱的"练习板"), 把烙铁烧热, 烙铁头同时接触焊盘 + 焊丝, 锡熔化后**先撤焊丝再撤烙铁**——形成一颗"小山丘"形状的焊点。**焊 10 个就熟了**。建议你焊一个简单图案 (一颗星 / 一个 SE), 当作"第一次焊接纪念章"贴在书桌前。

## 核心概念三: 5V → 3.3V 为什么不能直接连

回到开头那个烧 Pi 的故事。**Pi 的 GPIO 设计最大耐压只有 3.3V**, 接 5V 会烧。但很多传感器 (PMS5003 / HC-SR04 超声波 / DHT22 老版) 都需要 5V 供电、5V 输出信号。怎么办?

[[THEORY:theory_voltage_divider]]

最简单的办法: **电阻分压** (voltage divider)。两个电阻串联, 5V 接最上面, GND 接最下面, **中间抽一根线就是分压后的电压**。

[[IDEA:anim_voltage_divider_water]]

公式是: **V_out = V_in × R2 / (R1 + R2)**。

举个例子: 想从 5V 分到 3.3V, 选 **R1 = 1.7kΩ, R2 = 3.3kΩ** (实际买 1.8k + 3.3k 最接近):
- V_out = 5 × 3.3 / (1.8 + 3.3) = 5 × 3.3 / 5.1 = **3.24V** ✅ (Pi 能接受)

或者更傻瓜的组合: **R1 = R2 = 任意相等阻值** → V_out = 5 / 2 = 2.5V (低于 3.3V, 也安全, 但信号偏弱)。

[[IDEA:game_resistor_divider_lab]]

**分压电阻法适用于 "信号" 而不是 "供电"**——电阻分压输出电流很弱, 只够给 Pi GPIO 读个"高/低"信号, 不能给传感器供电。如果传感器需要 5V 供电, 5V 直接接传感器 VCC; 只是把传感器**输出的 DATA 信号**通过分压降到 3.3V 给 Pi 看。

**更好的方案: 买一个"电平转换器板" (logic level shifter)**——淘宝 3-5 块钱一块, 4 通道双向, 一端接 5V 设备一端接 3.3V Pi, 自动转换。M11 接 PMS5003 时我们会用这个, 这一节先理解概念 + 用电阻分压亲手验证。
